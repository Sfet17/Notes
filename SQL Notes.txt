------------------------------------------------------------------GROUP BY------------------------------------------------------------------------

 ->gropus by all rows by a specific column
 ->often used with aggregate functions like: AVG(),MAX(),MIN(),COUNT(),etc.
 ->WHERE clause dosen't work with GROUP BY, use HAVING
 
 table used for examples:
 								 transactions
						+------+--------+-------------+------------+
						| id   | amount | customer_id | date       |
						+------+--------+-------------+------------+
						| 1000 |  13.45 |           1 | 2023-01-12 |
						| 1001 |  21.43 |           2 | 2023-01-12 |
						| 1002 |  23.87 |           1 | 2023-01-12 |

	Examples:
							    SELECT MAX(amount),date
							    -> FROM transactions
							    -> GROUP BY date;
							+-------------+------------+

							+-------------+------------+
							|       23.87 | 2023-01-12 |
							|       12.97 | 2023-02-12 |
							+-------------+------------+
		


			   SELECT COUNT(amount),customer_id FROM transactions GROUP BY customer_id HAVING COUNT(amount) > 1;

							| COUNT(amount) | customer_id |

							|             2 |           1 |
							|             2 |           2 |
							+---------------+-------------+
 --ROLLUP--
  ->the sum of the GROUP BY clause

---------------------------------------------------------Rebuilding A Table--------------------------------------------------------------
 I. Create a new table with the same struture as another one---
	CREATE TABLE new_table LIKE old_table;
		-> the command creates the new_table that has the exaxct same columns, indexes and structure as the old_table;
		-> this command only copies the structure, not the actual data;
 II. Copy the data to the new table (with the same constraints)
 	INSERT INTO new_table(fields from the old table) SELECT fields from the old table FROM old_table [constraints];
 		-> 
 III. Remove the old_table
 	DROP TABLE old_table;
 IV. Rename the new table
 	RENAME TABLE new_table TO old_table;
 	
  Example:  	CREATE TABLE Users_new LIKE Users;
  		INSERT INTO Users_new (email, bio, country) SELECT email, bio, country FROM Users ORDER BY id;
  		DROP TABLE Users;
  		RENAME TABLE Users_new to Users;
